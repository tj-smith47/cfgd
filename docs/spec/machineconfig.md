# MachineConfig Spec Reference

`MachineConfig` is a namespaced Kubernetes custom resource (`cfgd.io/v1alpha1`) that represents the
desired configuration state for a single machine. The cfgd operator reads these resources and pushes
configuration to registered devices via the device gateway. The cfgd daemon on each machine also
reconciles against the `MachineConfig` assigned to it.

**API group:** `cfgd.io/v1alpha1`
**Scope:** Namespaced

## Document Structure

```yaml
apiVersion: cfgd.io/v1alpha1
kind: MachineConfig
metadata:
  name: string
  namespace: string

spec:
  hostname: string
  profile: string

  moduleRefs:
    - name: string
      required: bool

  packages:
    - string

  packageVersions:
    package-name: version-string

  files:
    - path: string
      content: string
      source: string
      mode: string

  systemSettings:
    key: value

status:
  lastReconciled: string
  driftDetected: bool
  observedGeneration: int

  conditions:
    - type: string
      status: string
      reason: string
      message: string
      lastTransitionTime: string
```

---

## Fields

### metadata

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `name` | string | Yes | | Resource name. Conventionally matches the device hostname or a `<team>-<username>` pattern when generated by the Crossplane composition. |
| `namespace` | string | Yes | | Kubernetes namespace. |

---

### spec

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `hostname` | string | Yes | | The hostname of the target machine. Must not be empty. |
| `profile` | string | Yes | | The cfgd profile name that should be active on this machine. Must not be empty. |
| `moduleRefs` | list | No | `[]` | Modules that should be installed on the machine. See [spec.moduleRefs[]](#specmodulerefs). |
| `packages` | list of string | No | `[]` | Additional packages to install beyond those declared in modules. |
| `packageVersions` | map | No | `{}` | Reported installed versions keyed by package name (e.g. `{"kubectl": "1.28.3"}`). Versions are loose semver: `1.28`, `1.28.3`. |
| `files` | list | No | `[]` | Files that should be present on the machine. See [spec.files[]](#specfiles). |
| `systemSettings` | map | No | `{}` | Arbitrary key/value system settings to enforce (e.g. sysctl values, OS defaults). |

---

### spec.moduleRefs[]

Each entry references a module that must be present on the machine.

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `name` | string | Yes | | Module name. Must match a `Module` resource name in the cluster or a local module in the device's config. |
| `required` | bool | No | `false` | When `true`, the reconciler treats a missing or failed module as a hard error that blocks the overall reconcile. When `false`, the module is applied on a best-effort basis. |

**Example:**
```yaml
moduleRefs:
  - name: kubectl
    required: true
  - name: k9s
    required: false
```

---

### spec.files[]

Each entry declares a file that should exist at a specific path on the machine.

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `path` | string | Yes | | Absolute destination path on the machine. Must not be empty. |
| `content` | string | No | | Inline file content. Exactly one of `content` or `source` must be set. |
| `source` | string | No | | Remote URL or git path from which to fetch the file. Exactly one of `content` or `source` must be set. |
| `mode` | string | No | `"0644"` | File permission bits in octal notation (e.g. `"0644"`, `"0755"`). Must be a valid octal string not exceeding `7777`. |

Validation enforces that every entry has `path` set and either `content` or `source` (not neither,
not both omitted).

**Example:**
```yaml
files:
  - path: /etc/containerd/config.toml
    content: |
      [plugins."io.containerd.grpc.v1.cri"]
        sandbox_image = "registry.k8s.io/pause:3.9"
    mode: "0644"
  - path: /etc/kubernetes/audit-policy.yaml
    source: https://raw.githubusercontent.com/acme/k8s-config/main/audit-policy.yaml
    mode: "0640"
```

---

### status

Written by the operator after each reconciliation pass. Do not set manually.

| Field | Type | Description |
|-------|------|-------------|
| `lastReconciled` | string (ISO 8601) | Timestamp of the last successful reconciliation. |
| `driftDetected` | bool | `true` if the last reconciliation found divergence between desired and actual state. |
| `observedGeneration` | int | The `metadata.generation` that was last processed by the controller. |
| `conditions` | list | Standard Kubernetes condition list. See [status.conditions[]](#statusconditions). |

---

### status.conditions[]

Follows the standard Kubernetes condition convention.

| Field | Type | Description |
|-------|------|-------------|
| `type` | string | Condition type identifier (e.g. `Reconciled`, `DriftDetected`, `Ready`). |
| `status` | string | `"True"`, `"False"`, or `"Unknown"`. |
| `reason` | string | Short CamelCase reason token. |
| `message` | string | Human-readable explanation. |
| `lastTransitionTime` | string (ISO 8601) | When this condition last changed status. |

---

## Full Example

```yaml
apiVersion: cfgd.io/v1alpha1
kind: MachineConfig
metadata:
  name: alice-workstation
  namespace: team-platform
spec:
  hostname: alice-mbp
  profile: work
  moduleRefs:
    - name: kubectl
      required: true
    - name: terraform
      required: false
  packages:
    - ripgrep
    - fd
  packageVersions:
    kubectl: "1.28.3"
    terraform: "1.6.0"
  files:
    - path: /etc/hosts.local
      content: "10.0.1.5  internal.acme.com\n"
      mode: "0644"
  systemSettings:
    net.ipv4.ip_forward: "1"
```
